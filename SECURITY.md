SECURITY

Security Policy

The Open Mission community takes security seriously.

If you discover a security vulnerability in any Open Mission project, please report it responsibly so we can investigate and address the issue before public disclosure.

⸻

Reporting a Vulnerability

Please avoid creating a public GitHub Issue for security vulnerabilities.

Instead, contact the project maintainers privately with as much information as possible, including:

* A description of the vulnerability
* Steps to reproduce the issue
* Potential impact
* Suggested mitigation (if available)

We will acknowledge your report as soon as possible and work toward a resolution.

⸻

Responsible Disclosure

We ask that you give the maintainers reasonable time to investigate and fix the issue before publicly disclosing it.

Responsible disclosure helps protect users and projects across the Open Mission ecosystem.

⸻

Supported Projects

Security updates are prioritized for actively maintained repositories.

Each repository may define additional security policies specific to that project.

⸻

Our Commitment

We appreciate everyone who helps improve the security of Open Mission.

Security is a shared responsibility, and responsible reporting helps make the ecosystem safer for everyone.
